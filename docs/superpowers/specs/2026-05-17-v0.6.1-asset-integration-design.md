- Rich region 的 Tooltip / drill-down 渲染 → 后续 spec
- 笔记 / 场景 / 内容块 的二次填充 → 用户后期补充

## 4. 转换器与加载器

### 4.1 `src/books/de-eu-vat/loader.ts`

入口:

```ts
export type BookData = {
  manifest: BookManifest;
  images: ImageAsset[];
  overlays: RichOverlayConfig[];
  glossary: GlossaryEntry[];
};

export async function loadDeEuVat(baseUrl = '/books/de-eu-vat/v0.6.1'): Promise<BookData>;
```

内部步骤:

1. `fetch(baseUrl + '/data/page_catalog.json')` → `RawPageCatalog`
2. 并行 `Promise.all` 加载 22 个 overlay JSON
3. 构建 ImageAssets:
   ```ts
   {
     assetId: `${sectionCode}-current-final-v06`,
     version: '0.6.1',
     src: `${baseUrl}/images/${pageId}_current_final.png`,
     width: 1086, height: 1448,
     alt: { 'zh-CN': entry.title },
   }
   ```
4. 转换 overlays:每个 raw overlay → `convertOverlay`
5. 构建 manifest:固定 bookId/slug + dynamic readingOrder + featureFlags
6. import 现有 `./glossary` 作为 glossary 直接附上

### 4.2 `convertOverlay(raw, sectionCode): RichOverlayConfig`

核心算法:

```ts
function convertOverlay(raw: RawOverlay, sectionCode: string): RichOverlayConfig {
  const { width, height } = raw.canvas;
  const toPct = (b: RawBBox): PercentageRect => ({
    x: (b.x / width) * 100,
    y: (b.y / height) * 100,
    width: (b.w / width) * 100,
    height: (b.h / height) * 100,
  });

  const regions: RichRegion[] = [
    ...raw.textRegions.map(r => buildRegion('textRegion', r, toPct)),
    ...raw.sections.map(r => buildRegion('section', r, toPct)),
    ...raw.gridRegions.map(r => buildRegion('gridRegion', r, toPct)),
    ...raw.imageHotspots.map(r => buildRegion('imageHotspot', r, toPct)),
    ...raw.navigationRegions.map(r => buildRegion('navigation', r, toPct)),
    ...raw.legalAnchors.map(r => buildRegion('legalAnchor', r, toPct)),
  ];

  // 投射到 Hotspot[](兼容现有 HotspotLayer)
  const hotspots: Hotspot[] = regions
    .filter(r => r.kind === 'navigation' || r.kind === 'imageHotspot' || r.kind === 'legalAnchor')
    .map(r => ({
      hotspotId: r.regionId,
      shape: 'rect',
      rect: r.rect,
      target: r.target ?? deriveTarget(r, sectionCode),
      label: { 'zh-CN': r.role },
      tooltip: r.text ? { 'zh-CN': r.text } : undefined,
    }));

  return {
    overlayId: `${sectionCode}-overlay-v06`,
    imageAssetId: `${sectionCode}-current-final-v06`,
    imageVersion: '0.6.1',
    coordinateSystem: 'percentage',
    canvas: { width, height },
    hotspots,
    regions,
  };
}
```

### 4.3 `deriveTarget(region, sectionCode): HotspotTarget`

- `navigation` 角色为 `'bottomNavigation'`:基于 region.role/sectionCode 推测下一页/上一页/TOC
  - 默认 `{ kind: 'page', pageId: 'toc' }`(回 TOC),不够精准可在后续迭代精修
- `imageHotspot`:default `{ kind: 'page', pageId: '<sectionCode>' }`(自身,no-op)
- `legalAnchor`:`{ kind: 'glossary' }`(暂跳转到术语表,Spec D 接 legal-source)

### 4.3b `buildRegion` 辅助

```ts
const KNOWN_COLOR_ROLES: ReadonlySet<ColorRole> = new Set([
  'evidence_green','tax_red','legal_purple','warning_amber','info_blue','neutral_slate',
]);

function buildRegion(kind: RegionKind, raw: RawRegion, toPct: (b: RawBBox) => PercentageRect): RichRegion {
  return {
    regionId: raw.id,
    kind,
    role: (raw.role as RegionRole),     // raw role 是 string,信任 schema(loader 测试覆盖)
    colorRole: raw.colorRole && KNOWN_COLOR_ROLES.has(raw.colorRole as ColorRole)
      ? (raw.colorRole as ColorRole) : undefined,
    rect: toPct(raw.bbox),
    text: raw.text,
    confidence: raw.confidence,
    source: raw.source,
  };
}
```

### 4.4 错误处理

- 单个 overlay fetch 失败 → 打 console.warn,该页 fallback 为 hotspots=[] + regions=[](仍可渲染图片)
- page_catalog fetch 失败 → throw,App 顶层 boundary 显示 EmptyState"图册加载失败"
- 校验 fields(canvas/textRegions/...)缺失 → 用默认空数组兜底,不 throw

## 6. App 异步初始化

### 6.1 改造前 vs 后

**当前 `App.tsx`** 顶层 const + 同步创建 registry。

**改造后** `App.tsx`(关键 import:`import { BookOpen, AlertTriangle } from 'lucide-react';` + `import { EmptyState, ToastProvider, TooltipProvider } from '../atlas-ui/primitives';` + `import { loadDeEuVat } from '../books/de-eu-vat/loader';` + `import { createBookRegistry } from '../atlas-core/registry';`):

```tsx
export function App() {
  const { pageId } = useParams<{ pageId?: string }>();
  const [registry, setRegistry] = useState<BookRegistry | null>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    loadDeEuVat()
      .then(data => setRegistry(createBookRegistry(
        data.manifest, data.images, data.overlays as unknown as OverlayConfig[],
        data.glossary, [], [], [], [], [],
      )))
      .catch(e => setError(String(e?.message ?? e)));
  }, []);

  if (error) {
    return <TooltipProvider><ToastProvider>
      <EmptyState icon={AlertTriangle} title="图册加载失败" description={error} />
    </ToastProvider></TooltipProvider>;
  }

  if (!registry) {
    return <TooltipProvider><ToastProvider>
      <EmptyState icon={BookOpen} title="加载图册中…" />
    </ToastProvider></TooltipProvider>;
  }

  return (
    <TooltipProvider delayDuration={200}>
      <ToastProvider>
        <MagazineReader registry={registry} initialPageId={pageId} />
      </ToastProvider>
    </TooltipProvider>
  );
}
```

### 6.2 测试影响

- 现有集成测试用 `createBookRegistry(...)` 直接构造 → 不变,仍可在测试里同步注入。
- `MagazineReader` 入参 `registry: BookRegistry` 保持不变 → 不需要改 reader 测试。
- App.tsx 没有单测(目前),如果加一个集成 smoke 测试需要 mock `fetch`。本期不强制。

## 9. 风险与应对

| 风险 | 应对 |
|---|---|
| 异步加载首屏空白 | EmptyState 加载态 + Suspense ready |
| 22 个 PNG 增大 build 体积 | Vite public/ 不打包到 dist js,按需 fetch;静态 URL 适合 HTTP cache |
| 旧 localStorage 评论 anchor 找不到新 pageId | 评论按 `currentPage.pageId` 过滤,失配 → 不显示;无害,无需迁移 |
| `notesDrawer = false` 会让 SlimRail 显示但 NotesTab 空 | SlimRail 不再显示 FileText 图标当 featureFlag false;或 NotesTab 在 noteIds 为空时显示空态(已有)。**实施时选择前者**(隐藏图标更干净) |
| Reader 测试用 `pages.ts` 静态 import | grep 找出后改用 fixture builder。文件不多 |
| convertOverlay 的 navigationRegion 推 target 不准 | 默认 `{ kind: 'page', pageId: 'toc' }` 安全 fallback;Spec D 精修 |
| Glossary 路由分离 | App.tsx 不再做检测,直接 router 切换 element |
| Vite 对 public/ 大量小 JSON 的并发 fetch | `Promise.all` 并发 22 个,现代浏览器 HTTP/2 ok |
